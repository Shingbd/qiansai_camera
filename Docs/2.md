# LVGL DRM / 触摸 / 显示旋转 问题记录

---

## 问题一：屏幕黑屏

**现象**：`lv_linux_drm_create()` 返回成功，但屏幕始终黑屏。

**根因**：`lv_linux_drm_create()` **只创建了 LVGL display 对象，未打开 DRM 设备**。必须调用 `lv_linux_drm_set_file()` 完成以下步骤：
1. 打开 `/dev/dri/cardN`
2. 查找 connector / plane / CRTC
3. 分配 dumb buffer 并 mmap
4. 通过 `lv_display_set_buffers_with_stride()` 设置 draw buffer

缺失该调用时，`drm_flush()` 中 `drm_dev->act_buf` 为 NULL，原子提交失败。

**修复**：
```c
char *drm_path = lv_linux_drm_find_device_path();
if (!drm_path) drm_path = strdup("/dev/dri/card0");
lv_linux_drm_set_file(disp, drm_path, -1);
lv_free(drm_path);
```

## 问题二：画面多层重叠 / 触摸回到桌面

**根因**：Xorg 运行在 VT7，占用 DRM primary plane 并读取 evdev。LVGL 的触摸事件被 Xorg 截获，导致「一触摸就回到桌面」。同时 DRM 配置仅设置了一个 plane，未关闭 primary plane 的旧画面，导致内容叠加。

**修复**：
- 切换到空闲 VT（如 VT3）运行，避开 Xorg：
  ```bash
  sudo chvt 3
  sudo ./build/qiansai model.rknn
  ```
- 使用 `EVIOCGRAB` 独占 evdev，防止 Xorg 读到触摸事件：
  ```c
  int fd = open(dev, O_RDWR | O_NOCTTY | O_CLOEXEC);
  ioctl(fd, EVIOCGRAB, 1);
  indev = lv_evdev_create_fd(LV_INDEV_TYPE_NONE, fd);
  ```

## 问题三：触摸无响应（VT3 下）

**现象**：切换到 VT3 后触摸无反应，日志显示 `evdev opened /dev/input/event0` 但坐标和光标均不动。

**根因**：初始化时用了 `LV_INDEV_TYPE_POINTER` 强制指定设备类型，但 `/dev/input/event0` 可能不是触摸设备（如 GPIO 键），导致 lvgl 直接按触摸设备创建 indev，实际收不到触摸事件。

**修复**：改用 `LV_INDEV_TYPE_NONE` 让 lvgl 自动检测设备类型，仅当检测类型为 `LV_INDEV_TYPE_POINTER` 时才使用该设备：
```c
indev = lv_evdev_create_fd(LV_INDEV_TYPE_NONE, fd);
if (indev && lv_indev_get_type(indev) != LV_INDEV_TYPE_POINTER) {
    lv_indev_delete(indev);
    continue;  // 不是触摸设备，试下一个
}
```

## 问题四：显示方向不对

**现象**：摄像头 800x480，LCD 物理分辨率 480x800，画面比例异常。

**方案 A（最终采用）**：在 NV12→RGB565 转换中做旋转，AI 推理用原始帧，不影响识别。

```c
// lvgl_ui.cpp
static int g_cam_rotation = 90;
static bool g_cam_mirror_h = true;

// nv12_to_rgb565() 中根据 rotation/mirror_h 映射像素坐标
```

**方案 B（备用）**：使用 `lv_display_set_rotation()` 旋转整个 LVGL 逻辑显示，但会影响 UI 控件布局。

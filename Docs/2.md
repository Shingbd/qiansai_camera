# LVGL DRM 初始化后屏幕黑屏 / 画面多层 / 触摸回桌面问题

> 共三个问题，依次排查修复。

---

## 问题一：屏幕黑屏

### 现象

`lv_linux_drm_create()` 返回成功（打印 "Display: 800x480"），但屏幕始终黑屏。

### 根因

`lv_linux_drm_create()` **只创建了 LVGL display 对象，未打开 DRM 设备**。必须调用 `lv_linux_drm_set_file()` 完成以下步骤：

1. 打开 `/dev/dri/cardN`
2. 查找 connector / plane / CRTC
3. 分配 dumb buffer 并 mmap
4. 通过 `lv_display_set_buffers_with_stride()` 设置 draw buffer

缺失该调用时，`drm_flush()` 中的 `drm_dev->act_buf` 为 NULL，原子提交失败，黑屏。

### 修复

```c
char *drm_path = lv_linux_drm_find_device_path();
if (!drm_path)
    drm_path = strdup("/dev/dri/card0");
lv_linux_drm_set_file(disp, drm_path, -1);
lv_free(drm_path);
```

## 问题二：画面有多层（重叠/残影）

### 根因

板子上运行了桌面环境（如 Weston），它占用了 DRM **primary plane** 并持续显示桌面内容。

`lv_linux_drm.c` 的 `find_plane()` 顺序查找第一个兼容的 plane，如果找到的是 **overlay plane**（而非 primary），则 primary plane 上的桌面内容仍然可见，导致两层画面叠加。

第一次 atomic commit 虽然做了 modeset（`DRM_MODE_ATOMIC_ALLOW_MODESET`），但**只配置了找到的那一个 plane**，其他 active plane 不会被自动禁用。

### 修复（二选一）

**方案 A：停止桌面环境**

```bash
# 查看当前哪个进程在用 DRM
ps aux | grep -E "weston|Xorg|wayland|sway"

# 停止 Weston（RK3588 常见）
sudo systemctl stop weston
# 或
sudo killall weston

# 确认 /dev/dri/card0 不再被占用
fuser /dev/dri/card0
```

**方案 B：切换到空闲 VT 运行**

```bash
# 先切换到 VT3（VT1/2 通常被桌面占用）
sudo chvt 3
# 然后运行程序
sudo ./build/qiansai model.rknn
```

## 问题三：触摸一下回到桌面

### 根因

桌面环境的输入栈（如 libinput）和 LVGL **同时打开了同一个 evdev 设备**。触摸事件被两方接收：
- LVGL 正常响应（点击按钮等）
- 桌面环境检测到输入后切换到自己的 VT，拿走显示

### 修复

使用 `EVIOCGRAB` 独占 evdev 设备，让桌面无法收到触摸事件：

```c
int fd = open("/dev/input/event0", O_RDWR | O_NOCTTY | O_CLOEXEC);
ioctl(fd, EVIOCGRAB, 1);
lv_indev_t *indev = lv_evdev_create_fd(LV_INDEV_TYPE_POINTER, fd);
```

通过 `lv_evdev_create_fd()`（而非 `lv_evdev_create()`）传入已 grab 的 fd。
